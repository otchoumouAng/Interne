Sécurisation & Déploiement Web

Ce guide transforme un serveur vierge en une machine sécurisée et prête pour la production avec Nginx, PostgreSQL, et un environnement Node.js pour servir des applications React/Vite.

Résumé de la Checklist

🚀 Phase 1 : Préparation & Accès

Avant même de vous connecter, préparez votre clé SSH sur votre ordinateur personnel.

Locale - Générer Clé 
ssh-keygen -t ed25519 -C "votre_email@example.com"
Locale - Déployer Clé 
ssh-copy-id root@IP_DU_VPS

🛠️ Phase 2 : Configuration du Système

1. Mise à jour initiale

apt update && apt full-upgrade -y

2. Création de l'utilisateur sudo

Règle d'or : Ne jamais travailler directement en root après cette étape. 
adduser adminuser
usermod -aG sudo adminuser

3. Fuseau horaire

timedatectl set-timezone Europe/Paris
timedatectl set-ntp true

🔐 Phase 3 : Durcissement SSH

Éditez /etc/ssh/sshd_config et modifiez les paramètres suivants :

  • Port 2222 (Changez le port par défaut)
  • PermitRootLogin no (Bloque root)
  • PasswordAuthentication no (Clés uniquement)
sudo sshd -t
sudo systemctl restart ssh

🔥 Phase 4 : Réseau & Protection

Pare-feu UFW

On ajoute Nginx Full pour ouvrir les ports 80 et 443.

sudo ufw default deny incoming
sudo ufw default allow outgoing
sudo ufw allow 2222/tcp
sudo ufw allow 'Nginx Full'
sudo ufw enable

Fail2ban

sudo apt install fail2ban -y
# Config dans /etc/fail2ban/jail.local
sudo systemctl restart fail2ban

🐘 Phase 5 : Base de données PostgreSQL

Installation du moteur de base de données relationnelle et création d'un utilisateur applicatif.

sudo apt install -y postgresql postgresql-contrib

# Créer un utilisateur et une DB
sudo -u postgres psql -c "CREATE USER app_user WITH PASSWORD 'MotDePasse123!';"
sudo -u postgres psql -c "CREATE DATABASE app_db OWNER app_user;"

🟩 Phase 6 : Environnement Node.js

Node.js est nécessaire pour exécuter le gestionnaire de paquets (npm) et construire (build) l'application React/Vite directement sur le serveur.

curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs
node -v && npm -v

🌐 Phase 7 : Nginx & Application React (Vite)

Installation du serveur web et création du Virtual Host (Bloc serveur). Notez la directive try_files essentielle pour le routage côté client d'une Single Page Application.

sudo apt install -y nginx
sudo mkdir -p /var/www/mon-domaine.com
sudo chown -R www-data:www-data /var/www/mon-domaine.com

# Création du fichier de config
sudo nano /etc/nginx/sites-available/mon-domaine.com

Contenu du fichier Nginx :

server {
    listen 80;
    server_name mon-domaine.com www.mon-domaine.com;

    root /var/www/mon-domaine.com/dist; # Dossier de build de Vite
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }
}
sudo ln -s /etc/nginx/sites-available/mon-domaine.com /etc/nginx/sites-enabled/
sudo rm /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl restart nginx

Script d'Automatisation (Tout-en-un)

Ce script exécute toutes les actions (Sécurité, PostgreSQL, Node, Nginx) en une seule passe. Éditez les variables au début du script avant de le lancer.

#!/bin/bash
set -e

# VARIABLES (À MODIFIER)
DOMAIN_NAME="ton-domaine.com"
APP_DIR="/var/www/$DOMAIN_NAME"
DB_NAME="app_db"
DB_USER="app_user"
DB_PASS="SuperPassword123!"

# 1. MISE À JOUR & UTILITAIRES
apt update && apt full-upgrade -y
apt install -y ufw fail2ban unattended-upgrades htop curl wget gnupg2

# 2. PARE-FEU & SSH
ufw default deny incoming
ufw allow 2222/tcp
ufw allow 'Nginx Full'
ufw --force enable

sed -i 's/^#Port 22/Port 2222/' /etc/ssh/sshd_config
sed -i 's/^#PermitRootLogin.*/PermitRootLogin no/' /etc/ssh/sshd_config
sed -i 's/^PasswordAuthentication yes/PasswordAuthentication no/' /etc/ssh/sshd_config
systemctl restart ssh
systemctl restart fail2ban

# 3. POSTGRESQL
apt install -y postgresql postgresql-contrib
sudo -u postgres psql -c "CREATE USER $DB_USER WITH PASSWORD '$DB_PASS';"
sudo -u postgres psql -c "CREATE DATABASE $DB_NAME OWNER $DB_USER;"

# 4. NODE.JS 20
curl -fsSL https://deb.nodesource.com/setup_20.x | bash -
apt install -y nodejs

# 5. NGINX & REACT CONFIG
apt install -y nginx
mkdir -p "$APP_DIR"
chown -R www-data:www-data "$APP_DIR"
chmod -R 755 "$APP_DIR"

cat <<EOF > /etc/nginx/sites-available/$DOMAIN_NAME
server {
    listen 80;
    server_name $DOMAIN_NAME www.$DOMAIN_NAME;
    root $APP_DIR/dist;
    index index.html;
    location / {
        try_files \$uri \$uri/ /index.html;
    }
}
EOF

ln -sf /etc/nginx/sites-available/$DOMAIN_NAME /etc/nginx/sites-enabled/
rm -f /etc/nginx/sites-enabled/default
systemctl restart nginx

echo "Installation terminée avec succès !"
